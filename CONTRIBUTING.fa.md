# مشارکت در پروژه مسابقات

اول از همه، از اینکه به فکر مشارکت در این پروژه هستید سپاسگزاریم! این افرادی مانند شما هستند که جامعه منبع باز را به مکانی عالی تبدیل می کنند.

## از کجا شروع کنم؟

اگر متوجه اشکالی شده اید یا درخواست ویژگی دارید، [یکی بسازید](https://github.com/your-username/your-repository/issues/new)! به طور کلی بهتر است قبل از شروع به کدنویسی، تأیید اشکال یا تأیید درخواست ویژگی خود را از این طریق دریافت کنید.

اگر سوال کلی دارید، در [بحث ها](https://github.com/your-username/your-repository/discussions) بپرسید.

## فورک و ایجاد یک شاخه

اگر فکر می کنید می توانید این مشکل را برطرف کنید، [مخزن را فورک کنید](https://github.com/your-username/your-repository/fork) و یک شاخه با نام توصیفی ایجاد کنید.

یک نام شاخه خوب (جایی که شماره مسئله 123 بلیطی است که روی آن کار می کنید) به این صورت خواهد بود:

```sh
git checkout -b 123-add-a-contributing-file
```

## اجرای مجموعه تست

برای اجرای تست ها، باید `docker` و `docker-compose` را نصب کرده باشید.

```sh
docker-compose up
```

این کار سرور توسعه را راه اندازی کرده و تست ها را اجرا می کند.

## پیاده سازی اصلاح یا ویژگی خود

در این مرحله، شما آماده ایجاد تغییرات خود هستید! در صورت نیاز به کمک، با خیال راحت بپرسید؛ همه در ابتدا مبتدی هستند :smile_cat:

برای سبک، ما از راهنمای سبک [PEP 8](https://www.python.org/dev/peps/pep-0008/) پیروی می کنیم. ما همچنین از `black` و `isort` برای قالب بندی کد استفاده می کنیم.

## ایجاد یک درخواست کشش

در این مرحله، باید به شاخه اصلی خود برگردید و مطمئن شوید که با آخرین نسخه بالادستی مخزن به روز است.

```sh
git remote add upstream git@github.com:your-username/your-repository.git
git checkout master
git pull upstream master
```

سپس شاخه ویژگی خود را از نسخه محلی master خود به روز کنید و آن را push کنید!

```sh
git checkout 123-add-a-contributing-file
git rebase master
git push --force-with-lease origin 123-add-a-contributing-file
```

در نهایت، به GitHub بروید و [یک درخواست کشش ایجاد کنید](https://github.com/your-username/your-repository/compare/master...123-add-a-contributing-file)

## به روز نگه داشتن درخواست کشش شما

اگر یک نگهدارنده از شما بخواهد که PR خود را "rebase" کنید، به این معنی است که کد زیادی تغییر کرده است و شما باید شاخه خود را به روز کنید تا ادغام آن آسان تر شود.

برای کسب اطلاعات بیشتر در مورد rebase و ادغام، [این راهنما](https://www.atlassian.com/git/tutorials/merging-vs-rebasing) را بررسی کنید.

## ادغام یک PR (برای نگهدارندگان)

یک PR فقط در صورتی می تواند توسط یک نگهدارنده ادغام شود که حداقل یک تأیید داشته باشد و تمام بررسی های وضعیت با موفقیت انجام شده باشد.

یک PR می تواند توسط یک نگهدارنده با استفاده از گزینه "Squash and merge" در شاخه `master` ادغام شود.
